# Couchbase Lite for Dart

Dart workspace monorepo using melos for orchestration. See [CONTRIBUTING.md](CONTRIBUTING.md)
for full setup instructions.

## Packages

| Package                         | Description                         |
| ------------------------------- | ----------------------------------- |
| `cbl`                           | Core Couchbase Lite library         |
| `cbl_sentry`                    | Sentry integration                  |
| `cbl_generator`                 | Code generation                     |
| `cbl_e2e_tests`                 | E2E test suite (not run directly)   |
| `cbl_e2e_tests_standalone_dart` | E2E test runner for standalone Dart |
| `cbl_e2e_tests_flutter`         | E2E test runner for Flutter         |
| `benchmark`                     | Performance benchmarks              |

## Development

### Formatting

- Run `daco format <file>` after editing Dart files.
- Run `clang-format` after editing C/C++ files.

### Analysis & Quality Gates

- `dart analyze --fatal-infos` ‚Äî run from repo root.
- `melos run quality-gates` ‚Äî formatting check + analyzer + unit tests.

### CLI Tool (`cbd`)

The project has a dev CLI in `tool/`. It is available on the PATH as `cbd`.

- Prefer `dart pub run cbd:cbd` over the globally installed `cbd`, which may be stale.
- `dart pub run cbd:cbd generate-bindings` ‚Äî regenerates FFI bindings.

## Auto-Generated Files ‚Äî Do Not Edit

The following files in `packages/cbl/lib/src/bindings/` are generated by ffigen ‚Äî do
not hand-edit them:

- `cblite.dart` / `cblitedart.dart`
- `cblite_symbols.yaml` / `cblitedart_symbols.yaml`

Regenerate with `dart pub run cbd:cbd generate-bindings`.

## Testing

### Unit Tests

Run from within the package directory:

```shell
cd packages/cbl && dart test
```

Unit tests exist for `cbl`, `cbl_generator`, and `cbl_sentry`.

### E2E Tests

1. Start Couchbase services: `melos run test:setupCouchbaseClusterDocker`
2. Run from `packages/cbl_e2e_tests_standalone_dart` without concurrency:
   ```shell
   cd packages/cbl_e2e_tests_standalone_dart && dart test --concurrency=1
   ```

The `cbl_e2e_tests/lib/src` directory is symlinked into
`cbl_e2e_tests_standalone_dart/test/cbl_e2e_tests`.

## Native Libraries & Build Hook

The build hook at `packages/cbl/hook/build.dart` handles native dependencies:

- Downloads precompiled **libcblite** (Couchbase Lite C SDK).
- Compiles **libcblitedart** from source (`packages/cbl/native/couchbase-lite-dart`).
- Optionally downloads **CouchbaseLiteVectorSearch** (enterprise, 64-bit only).

Edition and features are configured in the workspace root `pubspec.yaml` under
`hooks.user_defines.cbl`.

## Commits

Use [conventional commits](https://www.conventionalcommits.org/en/v1.0.0/). Scope to
the package when changes are limited to one, e.g. `feat(cbl): ...`.

## Pre-Commit Hooks (lefthook)

Automatically runs on staged files:

- `daco format` ‚Äî Dart formatting
- `dart fix --apply` ‚Äî auto-fixable issues
- `dart analyze --fatal-infos` ‚Äî analysis
- `clang-format` ‚Äî C/C++ formatting
- Prettier ‚Äî docs files
